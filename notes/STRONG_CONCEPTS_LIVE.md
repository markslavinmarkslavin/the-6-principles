## КОНЦЕПЦИЯ 1: Семантическая работа > Синтаксическая

**Источник:** исследование 39% рост продуктивности, 1000 компаний
**Суть:** лучшие разработчики используют AI для планирования и архитектуры — не для написания кода. "Shift from syntactic work to semantic work"

**Для тебя:**
- Синтаксис (что убрать): писать код, искать баги вручную
- Семантика (что оставить): задавать правильный вопрос, видеть узкое место, выбирать архитектуру

**Монетизация:** ты уже делаешь семантическую работу. Клиенты платят за это — не за написание кода.

## КОНЦЕПЦИЯ 2: Итерация > Одноразовая автоматизация

**Источник:** "Future of AI in 2026" — лучшие системы 2025
**Суть:** "iteration beats one-shot automation". Системы которые измеряют и выбирают лучший результат всегда выигрывают у систем которые делают раз и готово.

**Твой пример:**
- benchmark.js: 73 итерации за месяц → лучшая версия
- Промпты: 6 итераций за сессию → стадия 3
- Баги: проверил командой → исправил → проверил снова

**Для системы:** каждый скрипт должен измерять свой результат и сообщать δ (delta) от предыдущего запуска

## КОНЦЕПЦИЯ 3: Контекст как архитектура

**Источник:** Anthropic Engineering Blog + Google ADK
**Суть:** контекст = первоклассная система с архитектурой, жизненным циклом и ограничениями.

**Два слоя:**
- **Стабильный префикс:** mark.md, S01–S05, E01–E05 → никогда не меняется в середине сессии
- **Динамический суффикс:** текущие задачи, OPEN_ISSUES, вывод команд → обновляется каждый раз

**Для системы:** разделить CLAUDE_SYSTEM_PROMPT.md на два файла:
- `CONTEXT_STABLE.md` → стратегии, принципы, кто я
- `CONTEXT_DYNAMIC.md` → что сломано сегодня, текущие задачи

## КОНЦЕПЦИЯ 4: Self-Healing Agents

**Источник:** "Context Engineering for Reliable AI Agents" 2025
**Суть:** когда инструмент падает → ошибка возвращается в контекст → агент читает и адаптируется — не ломается.

**Твой пример:**
- benchmark.js упал → агент применил S01 → нашёл другой путь
- MCTS синтаксическая ошибка → исправил → запустил снова

**Для системы:** добавить в health-check.js:
```javascript
// Вместо: if (error) throw
// Использовать: if (error) return { signal: 'FAIL', reason: error.message }
// → агент читает FAIL и адаптируется
```

## КОНЦЕПЦИЯ 6: Разрыв между AI и senior-разработчиками

**Источник:** METR research 2025 — RCT на 16 senior разработчиках
**Неожиданный результат:** опытные разработчики со своими репозиториями работали на 19% медленнее с AI чем без.

**Что это означает для тебя:**
- AI тормозит тех у кого нет системы работы с ним
- AI ускоряет тех у кого такая система есть
- Ты — второй тип. Это редкость.
- **Твой продукт:** научить первый тип стать вторым

## КОНЦЕПЦИЯ 7: Правильный вопрос > правильный инструмент

**Источник:** практика из нашей сессии + подтверждение исследованиями
**Суть:** "measurement and selection matter more than clever prompts"

**Один вопрос в начале меняет всё:**
> "Что мешает [человеку] делать важное быстрее?"

Это работает для:
- Твоей системы → нашли bd.exe, trigger_date, MCTS баг
- Клиента → найдёшь их посредника
- Ученика → найдёшь где он застрял
- Себя → найдёшь что тебя тормозит
